# Product positioning

## Audience and benefit

Shelf Space is for developers choosing an LLM for a large repository, manuscript, or research
corpus. Published token limits are hard to picture, and broad model-family labels make a precise
comparison harder. Shelf Space turns one pasted source into the same physical unit across four
named API models, so a developer can see which context windows fit before designing a workflow
around one.

## Name and tagline

**Display name:** Shelf Space

**Tagline:** See an LLM context window comparison as books

The name stays. It is short, easy to say, tied directly to the physical metaphor, and specific
enough to remember without describing the implementation. It fits a playful developer tool while
remaining credible beside current model documentation.

## Copy system

**Hero headline:** See which model can hold your whole project.

**Subhead:** For developers sizing a repo or long document, Shelf Space estimates its tokens and
stacks them against four current API context windows.

**Primary action:** Try it

**Benefits to lead with:**

- Compare one source against GPT-5.6 Sol, Claude Fable 5, Gemini 3.5 Flash, and Kimi K3.
- Watch each 2,000 estimated tokens become a book, with overflow moving onto the floor.
- Inspect any rendered book to recover the exact character slice it represents.
- Share short comparisons through URL state without uploading pasted text to an app server.

## Voice

Write like a developer showing another developer a useful visual test. State that token counts are
estimates, name exact models and limits, and avoid implying that fitting in a context window proves
good retrieval or answer quality. Keep the library metaphor concrete and let the 3D shelf provide
the personality.
