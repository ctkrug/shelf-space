    A context limit you can picture
    
        
    See which model can hold your whole project.
    
        
    
          Shelf Space is an LLM context window comparison built for developers
          sizing a large repo, manuscript, or research corpus. Paste your text and it becomes a
          row of 2,000-token books on shelves sized to four current API models.
        
    
        View source on GitHub
      
    

      
    
        
    
          
    Compare named models
    
          
    
            The shelves use GPT-5.6 Sol, Claude Fable 5, Gemini 3.5 Flash, and Kimi K3 instead of
            vague vendor families. Published input limits are checked against vendor docs.
          
    
        
    
        
    
          
    Keep the scale physical
    
          
    
            Every book represents 2,000 estimated tokens. A one-million-token window holds 500
            books, so a full shelf and a floor spill have a consistent meaning.
          
    
        
    
        
    
          
    Inspect the source
    
          
    
            Click any book to open the exact character slice behind it. Short inputs can be shared
            through URL state; that encoding is not encryption, so leave sensitive text out.

    Direct answers
    
        
    Context window FAQ
    

        
    
          What does an LLM context window comparison show?
          
    
            It compares how many input tokens each named model can accept in one request. Shelf
            Space estimates one source four ways, then shows its share of each published window.
            It does not measure response quality, retrieval accuracy, price, or usable output room.
          
    
        
    
        
    
          How much text fits in Claude context window?
          
    
            Claude Fable 5 has a one-million-token context window according to Anthropic's current
            API documentation. The amount of text varies by content, and prompts plus expected
            output also consume the request budget.
          
    
        
    
        
    
          What is the Kimi K3 context window size?
          
    
            Kimi K3 supports up to 1,048,576 tokens through the API. Some Kimi membership tiers
            expose smaller limits, so check the plan and client you intend to use.
          
    
        
    
        
    
          Does fitting in the window mean the model will use every detail?
          
    
            No. A context limit is a hard capacity, not a promise of perfect recall. Long-context
            retrieval can weaken as prompts grow, and production work may still benefit from
            chunking, retrieval, caching, or a focused subset of the source.
